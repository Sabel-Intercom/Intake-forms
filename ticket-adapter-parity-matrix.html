Baseline recommendation

ARCHITECTURE / CODE BASELINE

Zendesk

Cleanest, most-complete Protocol triplet. Already the matrix-designated canonical reference for metric-cards + review/audit. Clone its sources/ triplet + wizard for any new protocol adapter (incl. FreshService).

FEATURE / UX BASELINE

Freshdesk

Richest feature set — defines wizard step order, the review/audit panel (renderStep13), attribute-driven inbox routing, sample/prod stage-DB split. But on the legacy bespoke stack — copy its features, not its architecture.

Parity blockers & metric-card divergences to normalise

  • Attribute-driven inbox routing — FD-only, in legacy tickets/; lift into shared core.
  • Full sample/prod stage-DB isolation — FD has stage-in-PK; protocol adapters share one DB.
  • Freshdesk's bespoke-stack debt — blocks FD from sharing features instead of forking them.
  • FreshService does not exist — empty branch + API docs skill only. Greenfield from the ZD baseline.
  • Metric-card label drift — success card is "Extracted"/"Loaded"/"OK"/"LOADED" depending on adapter+phase; Total↔PENDING↔Discovered for the same slot; clickable drill-in only on FD/HS. A shared card component in wizard_shell.js would normalise labels + give every adapter the throughput, cumulative and warning grids that only FD/HS/ZD have today.
